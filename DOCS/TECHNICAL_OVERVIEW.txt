KUKUTalk — Technical Overview (updated 2025-09-04)

Purpose
- Safe chat for children (role CHILD) with moderation and guardian/school oversight.

Architecture
- Backend: NestJS + Prisma (PostgreSQL), Redis for refresh token whitelist
- Frontend: React/Vite (apps/web)
- Runtime: Docker Compose
- Services: postgres, redis, api, guard (reverse-proxy with moderation + rate-limit)

Auth (E2E working)
- Endpoints: POST /auth/login, POST /auth/refresh, POST /auth/logout, GET /auth/me
- AccessToken (JWT) + RefreshToken (rotating); refresh tokens validated against Redis
- Redis URL (in containers): redis://redis:6379/0

Conversations & Messages (current)
- GET /conversations → ok
- GET /conversations/:id/messages → ok
- POST /conversations/:id/messages → ok (requires membership once enforced)
- POST /conversations → not implemented yet (temporary seeding via Prisma in dev)
- Minimal required Conversation fields (deduced from DMMF in dev): createdById, type ('DIRECT' works); name/title is not part of schema (server derives display elsewhere)

Guard service (moderation & rate-limit)
- Location: infra/docker/guard
- Exposed on host: http://localhost:3002 (proxy to api:3001)
- ENV: TARGET=http://api:3001, RL_MAX=5, RL_WIN=10000 (ms)
- Rules (Polish heuristics):
  • PII_PHONE, MEETUP_RISK → BLOCKED (synthetic response, nothing hits API)
  • BULLYING → FLAGGED (request forwarded; response status forced to FLAGGED)
- Rate-limit: 5 messages / 10s per (user×conversation). Excess → HTTP 429 (tests map to RATE_LIMITED).

Dev / Ops
- Compose base stack:
  docker compose -f infra/docker/docker-compose.yml up -d --build
- Guard with override:
  docker compose -f infra/docker/docker-compose.yml -f infra/docker/docker-compose.guard.yml up -d --build guard
- PowerShell Invoke-WebRequest gotchas:
  • Disable Expect: 100-continue in session
  • Do not force 'Connection: close' header; optionally use -DisableKeepAlive if available
- Backups (pit stop):
  pg_dump custom format to /backups/pg; backups ignored by git
- Support bundle: packaged logs/config for handoff; latest zip captured at creation time in user's workspace
